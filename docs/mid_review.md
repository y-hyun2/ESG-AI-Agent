## 📌 3. 주요 개발 항목별 상세 점검

### ✅ 3.1 데이터 처리 파이프라인

- PDF → Text 추출(PyPDF, Tesseract)  
- Text Chunking 적용  
- 벡터 임베딩(bge-m3) 정상 생성  
- ChromaDB에 저장 완료  

**상태:** 정상  
**향후 작업:** 데이터 정합성 점검 자동화 필요

---

### 🧠 3.2 RAG 검색 품질 점검

- 정책 문서 검색 정확도: 약 85% (K-ESG, GRI 문서 모두 활용)  
- 각 툴별로 retriever 설정/프롬프트가 달라 용도별 정확도 확보  
- 향후 MultiRetriever(다중 컬렉션) 구조로 확장 계획  

**상태:** 양호  
**개선 필요:** 문서별 가중치 조정(MMR), 다중 문서 병합

---

### 📑 3.3 정책 분석 모듈

| 항목 | 상태 | 코멘트 |
|------|--------|-----------|
| 요약 기능 | ✔ 정상 동작 | 결과 일관성 양호 |
| 비교 기능 | ▲ 일부 미흡 | 용어 매핑 고도화 필요 |
| K-ESG ↔ SASB 매핑 | 진행 중 | 정규 표현식 기반 자동화 |

**핵심 이슈:** 근거 문서/신뢰도 표기 및 숫자 출처 명시 강화 필요

---

### ⚠️ 3.4 리스크 진단 모듈

- ISO 31000 / Materiality / Supplier 평가를 LangGraph 파이프라인에 통합  
- 체크리스트·보고서·CSV/XLSX/PDF 자동 출력 구현  

**이슈:** 현장 유형별 템플릿 확장, MultiRetriever 연구 필요  

---

### 📄 3.5 보고서 생성 모듈

- ReportTool 폴더 버전 통합: `generate_report_from_query`로 HTML 보고서 생성  
- K-ESG 61항목 템플릿 + SDGs/ISO 26000 매핑 구성  

**이슈:** 예시 데이터 세트 확장, 프롬프트 기반 요약 자동화

---

### 📡 3.6 규제 모니터링 모듈

- Selenium + Tavily + Chroma로 최신 규제 요약 자동 수집  
- LangGraph 파이프라인과 통합하여 챗봇에서도 자동 활용  

**상태:** 진행 중  
**이슈:** EU 규제 등 다중 소스 병합 설계 예정

---

### 🖥 3.7 React UI

- Moonlight 3분할 레이아웃 (사이드바/중앙/챗봇) 완성  
- 챗봇은 `react-markdown`, SSE 스트리밍, 엔터 전송 기능 등 도입  

**이슈:** 대규모 텍스트 렌더링 최적화, 그래프 시각화 추가
